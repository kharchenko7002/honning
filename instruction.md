# Instruction for Claude

You are helping me build a simple, clean, school project website from scratch.

## Project type
Build a small responsive website for selling honey.

The website should feel like a real small local honey business, but it does not need full e-commerce functionality.
Keep it realistic, simple, easy to understand, and easy to explain to a teacher.

## Main goal
Create a pleasant and well-structured website where a user can:
- understand what the business sells
- view different types of honey
- learn about the business
- find contact information
- find where we are located
- send a simple order/request

## Important priority
This is a school project, so:
- the code must be clean and understandable
- the structure must be simple
- do not overcomplicate the project
- avoid unnecessary advanced features
- prefer a polished basic version over a complex unfinished version

## Language
Use Norwegian Bokmål for all visible website text.

## Design style
The design should use soft, warm, natural colors inspired by:
- honey
- beeswax
- nature
- light brown / beige / warm yellow tones

The design should feel:
- friendly
- calm
- local
- trustworthy
- easy to use

## Required sections / pages
The website must include these main parts:

1. **Home page**
   - hero section with title, short description, and call-to-action button
   - short presentation of the honey business
   - featured honey products
   - button to view all honey types

2. **Honey / Products section**
   - show several different kinds of honey
   - each product should have:
     - name
     - short description
     - price
     - image or placeholder image
     - small label such as mild / strong / floral / forest honey
   - allow the user to browse the honey selection easily

3. **About us**
   - explain who we are
   - explain where the honey comes from
   - explain why the honey is special

4. **Where we are**
   - simple location section
   - address or fictional local area
   - opening times
   - optional small embedded map placeholder or map section layout

5. **Contact**
   - contact form with:
     - name
     - email
     - message
   - business contact information
   - phone
   - email
   - address

6. **Navigation**
   - main menu at the top
   - links to:
     - Hjem
     - Honning
     - Om oss
     - Hvor finner du oss
     - Kontakt
   - mobile-friendly navigation

## Functional expectations
The website should include basic working interactivity where appropriate:
- navigation links should work
- buttons should scroll or route correctly
- contact form should validate basic input
- honey cards should be clearly displayed
- product section should be easy to read
- layout must work on mobile and desktop

## Technical requirements
Use a simple frontend stack.

Preferred option:
- HTML
- CSS
- JavaScript

If you think React is better, only use it if the structure stays simple and easy to explain.

## Code quality requirements
- use semantic HTML
- keep CSS organized and readable
- keep JavaScript minimal and clear
- comment important code sections
- use good file naming
- do not add unnecessary libraries
- keep the project beginner-friendly

## Accessibility and usability
Please make the site reasonably accessible:
- good contrast
- readable font sizes
- clear button text
- alt text for images
- labels for form fields
- consistent layout

## Content tone
The text should sound like a small Norwegian local honey producer.
It should feel warm, natural, and trustworthy.

## Suggested honey products
Include around 4–6 example honey products such as:
- Blomsterhonning
- Skogshonning
- Lynghonning
- Sommerhonning
- Fjellhonning
- Kremhonning

Each product can have a short fictional description.

## Keep the scope realistic
Do NOT build:
- login system
- payment integration
- advanced backend
- database
- admin dashboard
- shopping cart with checkout

Instead, if needed, simulate ordering with:
- a contact form
- a simple “Bestill nå” button
- a request form

## What I want from you
Please work in this order:

1. First, give a short project plan
2. Then suggest a simple folder structure
3. Then create the full code
4. Then explain how each file works
5. Then suggest 3–5 realistic future improvements for the next school phase

## Output format
When generating the project:
- clearly separate each file
- show file names
- provide complete code
- make sure everything matches
- avoid half-finished snippets

## Extra note
This project should be easy to use later for:
- user stories
- actors
- use case
- use case diagram
- reflection about improvements

So build the site in a way that makes it easy to explain user needs and future improvements.
